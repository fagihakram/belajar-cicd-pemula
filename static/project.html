CI/CD Pipeline

Continuous Integration (CI) adalah pengintegrasian kode ke dalam repositori kode kemudian menjalankan pengujian secara otomatis, cepat, dan sering. Kamu dapat melakukan CI ini dengan menggunakan perintah commit.

Sementara Continuous Delivery atau Continuous Deployment (CD) adalah praktik yang dilakukan setelah proses CI selesai dan seluruh kode berhasil terintegrasi, sehingga aplikasi bisa dibangun lalu dirilis secara otomatis. CI/CD pipeline ini sangat lazim digunakan dalam pengembangan perangkat lunak. CI/CD pipeline ini menjadi penghubung antara tim pengembang dengan tim operasional yang di dalamnya terdapat tiga fase yang berupa continuous integration, continuous delivery, dan continuous deployment. Ketiga fase tersebut akan dilakukan secara terus menerus dan otomatis untuk mendapatkan perangkat lunak yang andal dan bebas dari bug.

Praktikum 4 POPL

Pada praktikum kali ini kita akan mempraktekan CI/CD Pipeline dengan GitHub dan Docker.

Beberapa hal yang perlu dipersiapkan terlebih dahulu yaitu:

  • Registrasi akun DockerHub: https://hub.docker.com/
  • Install Docker Desktop

Ikuti langkah-langkah dibawah:

  1. Fork repository Github.com/Maulyanda/belajar-cicd-pemula
  2. Pada repository pribadi hapus directory “.github/workflows”
  3. Buka halaman actions pada GitHub
  4. Pilih Go lalu Configure
  5. Commit dengan membuat branch baru
  6. Create Pull Request lalu Merge Pull Request
  7. Jika sudah berhasil maka tampilan akan seperti dibawah
  8. Pergi ke Details untuk melihat hasil build
  9. Kita dapat melihat keseluruhan proses build yang terjadi sehingga mempermudah kita dalam proses debugging
  10. Login ke Docker Hub lalu buat repository baru
  11. Jika repo pada Docker Hub sudah terbuat, selanjutnya adalah membuat personal access token
  12. Pada sisi kanan atas tekan gambar profil lalu pergi ke Account settings
  13. Lalu pilih Personal access tokens
  14. Kemudian generate access token dengan deskripsi “DOCKERHUB_TOKEN” dan access permissions “Read, Write, Delete”
  15. Kembali ke GitHub
  16. Pada repository masing-masing buka settings lalu ke bagian Actions di dalam Secret and variables
  17. Buat 2 secret key dengan menekan “new repository secret”:
    • Name: DOCKERHUB_USERNAME
      Secret: (username pada Docker Hub)
    • Name: DOCKERHUB_TOKEN
      Secret: (personal access token yang sudah dibuat pada Docker Hub sebelumnya)
  18. Jika sudah berhasil maka tampilan akan seperti ini
  19. Sekarang kita akan mengkoneksikan Docker, buka “go.yml” pada directory “.github/workflows”
  20. Tambahkan code ini ke dalam file “go.yml”
  21. - Ganti “akhyarrrrr/praktikum-popl” dengan nama repository yang telah dibuat pada Docker Hub
  22. - Perhatikan indentasinya karena ini bersifat sensitif
  23. Lalu commit dan push perubahan yang sudah dibuat
  24. Sekarang kita akan menjalankan project kita pada Docker Desktop
  25. Buka Docker Desktop yang sudah terinstal pada laptop masing-masing
  26. Masuk ke bagian Hub di dalam Images kemudian pull
  27. Setelah berhasil pull, maka tampilan pada Local akan seperti ini
  28. Jalankan Docker dengan port 3000
  29. Jika project berjalan dengan lancar maka tampilan pada http://localhost:3000 akan seperti ini.
